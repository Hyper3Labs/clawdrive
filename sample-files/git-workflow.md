# Git Workflow Guide

## Branch Naming
- `feat/` — new features
- `fix/` — bug fixes
- `refactor/` — code restructuring
- `docs/` — documentation updates

## Commit Message Format
```
type(scope): short description

Longer explanation if needed.
```

## Review Checklist
- [ ] Tests pass locally
- [ ] No lint errors
- [ ] Types are correct
- [ ] Edge cases handled
